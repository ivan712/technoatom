	</header><!-- .entry-header -->

    

	
	            <div class="post-image">
            <img width="1118" height="631" src="https://python-scripts.com/wp-content/uploads/2019/10/Requests-Python-1.jpg" class="wp-image wp-post-image" alt="Requests Python" itemprop="image" srcset="https://python-scripts.com/wp-content/uploads/2019/10/Requests-Python-1.jpg 1118w, https://python-scripts.com/wp-content/uploads/2019/10/Requests-Python-1-300x169.jpg 300w, https://python-scripts.com/wp-content/uploads/2019/10/Requests-Python-1-768x433.jpg 768w, https://python-scripts.com/wp-content/uploads/2019/10/Requests-Python-1-1024x578.jpg 1024w" sizes="(max-width: 1118px) 100vw, 1118px" />        </div>
    	
	<div class="entry-content" itemprop="articleBody">
		<p>Библиотека <code>requests</code> является стандартным инструментом для составления HTTP-запросов в Python. Простой и аккуратный API значительно облегчает трудоемкий процесс создания запросов. Таким образом, можно сосредоточиться на взаимодействии со службами и использовании данных в приложении.</p>
<h3>Содержание статьи</h3>
<ul>

<li><a href="#header-modify-requests">Настройка HTTP заголовка запроса (headers)</a></li>
<li><a href="#examples-methods-requests">Примеры HTTP методов в Requests</a></li>
<li><a href="#message-body">Python Requests тело сообщения</a></li>

</ul>
<p>В данной статье представлены наиболее полезные особенности requests. Показано, как изменить и приспособить requests к различным ситуациям, с которыми программисты сталкиваются чаще всего. Здесь также даются советы по эффективному использованию requests и предотвращению влияния сторонних служб, которые могут сильно замедлить работу используемого приложения. Мы использовали библиотек requests в уроке по парсингу html через библиотеку <a href="https://python-scripts.com/beautifulsoup-html-parsing">BeautifulSoup</a>.</p>
<p><strong>Ключевые аспекты</strong> инструкции:</p>
<ul>
<li><strong>Создание запросов</strong> при помощи самых популярных HTTP методов;</li>
<li><strong>Редактирование</strong> заголовков запросов и данных при помощи строки запроса и содержимого сообщения;</li>
<li><strong>Анализ</strong> данных запросов и откликов;</li>
<li>Создание <strong>авторизированных</strong> запросов;</li>
<li><strong>Настройка</strong> запросов для предотвращения сбоев и замедления работы приложения.</li>
</ul>
<p>В статье собран оптимальный набор информации, необходимый для понимания данных примеров и особенностей их использования. Информация представлена в доступной в форме. Тем не менее, стоит иметь в виду, что для оптимального разбора инструкции потребуются хотя бы <a href="https://ru.wikipedia.org/wiki/HTTP" target="_blank" rel="nofollow noopener">базовые знания HTTP</a>.</p>
<p>Далее будут показаны наиболее эффективные методы использования <code>requests</code> в разрабатываемом приложении.</p>

<p>Заголовок <code>Accept</code> сообщает серверу о типах контента, который можно использовать в рассматриваемом приложении. Здесь подразумевается, что все совпадения будут подсвечены, для чего в заголовке используется значение <code>application/vnd.github.v3.text-match+json</code>. Это уникальный заголовок <code>Accept</code> для GitHub. В данном случае содержимое представлено в специальном <strong>JSON формате</strong>.</p>

<h2 id="inspect-ssl">Python Requests проверка SSL сертификата</h2>
<p>Всякий раз, когда данные, которые вы пытаетесь отправить или получить, являются конфиденциальными, безопасность важна. Вы общаетесь с защищенными сайтами через <strong>HTTP</strong>, устанавливая зашифрованное соединение с использованием <strong>SSL</strong>, что означает, что проверка <strong>SSL сертификата</strong> целевого сервера имеет решающее значение.</p>
<p>Хорошей новостью является то, что <code>requests</code> по умолчанию все делает сам. Однако в некоторых случаях необходимо внести определенные поправки.</p>
<p>Если требуется <strong>отключить проверку SSL-сертификата</strong>, параметру <code>verify</code> функции запроса можно присвоить значение <code>False</code>.</p><!-- Crayon Syntax Highlighter v_2.7.2_beta -->

<h2 id="app-performance">Python Requests производительность приложений</h2>
<p>При использовании <code>requests</code>, особенно в среде приложений, важно учитывать влияние на производительность. Такие функции, как <strong>контроль таймаута</strong>, сеансы и ограничения повторных попыток, могут помочь обеспечить бесперебойную работу приложения.</p>
<h3>Таймауты</h3>
<p>Когда вы отправляете встроенный запрос во внешнюю службу, вашей системе нужно будет дождаться ответа, прежде чем двигаться дальше. Если ваше приложение слишком <strong>долго ожидает ответа</strong>, запросы к службе могут быть сохранены, пользовательский интерфейс может пострадать или фоновые задания могут зависнуть.</p>
<p>По умолчанию в requests на ответ время не ограничено, и весь процесс может занять значительный промежуток. По этой причине вы всегда должны <strong>указывать время ожидания</strong>, чтобы такого не происходило. Чтобы <strong>установить время ожидания запроса</strong>, используйте параметр <code>timeout</code>. <code>timeout</code> может быть целым числом или числом с плавающей точкой, представляющим количество секунд ожидания ответа до истечения времени ожидания.</p><!-- Crayon Syntax Highlighter v_2.7.2_beta -->

<h2 id="object-session">Объект Session в Requests</h2>

<p>Каждый раз, когда вы делаете запрос <code>session</code>, после того как он был инициализирован с учетными данными аутентификации, учетные данные будут сохраняться.</p>
<p>Первичная оптимизация производительности сеансов происходит в форме постоянных соединений. Когда ваше приложение устанавливает соединение с сервером с помощью <code>Session</code>, оно сохраняет это соединение в пуле соединений. Когда ваше приложение снова хочет подключиться к тому же серверу, оно будет использовать соединение из пула, а не устанавливать новое.</p>

<p>Изучение библиотеки <strong>Python requests</strong> является очень трудоемким процессом.</p>
<p>После разбора данных в статье примеров можно <strong>научиться тому, как</strong>:</p>
<ul>
<li>Создавать запросы, используя различные <strong>методы HTTP</strong> – <code>GET</code>, <code>POST</code> и <code>PUT</code>;</li>
<li>Настраивать свои запросы, изменив заголовки, аутентификацию, строки запросов и тела сообщений;</li>
<li>Проверять данные, которые были отправлены на сервер, а также те данные, которые сервер отправил обратно;</li>
<li>Работать с <strong>проверкой SSL сертификата</strong>;</li>
<li>Эффективно использовать <code>requests</code>, <code>max_retries</code>, <code>timeout</code>, <code>Sessions</code> и транспортные адаптеры.</li>
</ul>
<p>Грамотное использование <strong>requests</strong> позволит наиболее эффективно настроить разрабатываемые приложения, исследуя широкий спектр веб-сервисов и данных, опубликованных на них.</p>
